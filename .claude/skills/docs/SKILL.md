# /docs - Documentation Workflow

**Purpose**: Create or update documentation. No code changes.

**Key Characteristics**:
- Only .md, .txt, .json, .yaml file edits allowed
- No agents required
- No task tracking required
- For READMEs, architecture docs, API docs, guides

---

## When to Use /docs

- Writing or updating README files
- Documenting architecture decisions
- Creating API documentation
- Writing developer guides
- Updating configuration docs

---

## Workflow

### Step 1: UNDERSTAND SCOPE

What documentation needs to change and why.

### Step 2: WRITE

Create or edit documentation files directly.

### Step 3: VERIFY

- Links are correct
- Code examples are accurate
- No stale references

---

## What's Allowed

- Edit/Write on .md, .txt, .json, .yaml, .yml, .toml files
- Read any file (for reference)

## What's Blocked

- Edit/Write on code files (.py, .ts, .tsx, .js, etc.)
- Agent invocation (not needed for docs)
